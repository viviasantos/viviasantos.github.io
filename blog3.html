⚡ How to Structure Scrum Ceremonies for Agile and Valuable Deliveries ⚡

    I want to share my experience implementing and leading Scrum ceremonies in short 2-week sprints. One of the biggest lessons I learned in practice is the importance of breaking down deliveries into small pieces that already add value to the client, so they don’t have to wait long and can quickly see real benefits. Besides that, using Scrum ceremonies brings clear gains: it improves team alignment, helps identify and mitigate risks, enhances communication, and keeps the focus on what really matters — delivering continuous and sustainable value.

    1. Sprint Planning: Planning with Scrum Poker and Business Alignment
      At the start of each sprint, I organize a planning meeting where we use Scrum Poker 🃏 to estimate tasks. This technique helps the team discuss and reflect on the complexity of each backlog item, promoting more accurate and collaborative estimates.
      Before this meeting, I do a prior alignment with the business team 🤝 to understand risks ⚠️, urgency ⏰, and impact 🚀 of the requests. This ensures we prioritize what really matters to the client and the business, and the team is aware of challenges to better plan the work.
      Talking with the team at this point is essential to clear doubts 💬, identify dependencies 🔗, and understand the effort needed for each delivery.
    🛠 Tools & Technologies Used:
      🔗 ServiceNow REST API – For accessing incident data.
      📡 REST API Explorer – For building and testing the endpoint.
      🐍 Python 3 – Core scripting language.
      📦 requests, pandas – requests for API communication/pandas for data manipulation/export.
      💻 Visual Studio Code – IDE for development.
      📁 Output: .xlsx file containing incident records.

ServiceNow to Jira Integration via REST Message (P1 Incidents)

    Description: This ServiceNow project uses a REST Message to send P1 incidents to an external system (like Jira). The data is sent as JSON to a simulated endpoint (Webhook.site) when a critical incident is created.

    🚀 What This Project Does:
      - Sends incident data from ServiceNow to an external system using REST POST.
      - Triggered automatically when a new P1 incident is inserted.
      - JSON body includes incident number, short description, and priority.
      - Tested using Webhook.site as a mock endpoint.
    🛠 Tools & Technologies Used:
      🔗 ServiceNow REST Message – For sending REST POST requests.
      ⚙️ Business Rule – Executes on After Insert for new incidents.
      🌐 Webhook.site – For receiving and testing the payload.
      💡 Variable substitution – Dynamic data injection into JSON body.
      🖥️ ServiceNow Studio – Development and configuration.
