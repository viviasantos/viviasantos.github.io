Implementation of Scrum in IT Governance with ServiceNow

Description: This project guides you through the implementation of the Scrum methodology in the IT Governance area using the ServiceNow platform. It was designed to organize and accelerate the delivery of improvements and automations, while promoting agility, collaboration, and continuous value delivery through structured sprints.
🚀 What This Project Teaches:
    - How to raise awareness and train your team in agile practices using Scrum.
    - How to map and model demand workflows in ServiceNow for better visibility.
    - How to structure and prioritize a product backlog using user stories.
    - How to define clear acceptance criteria for development requests.
    - How to organize and facilitate Scrum ceremonies (Planning, Daily, Review, Retrospective).
    - How to create sprint dashboards to track KPIs such as velocity and lead time.
    - How to collaborate with the Product Owner for continuous backlog refinement.
📊 Results Achieved:
    - 📉 40% reduction in time to deliver platform improvements.
    - 🤝 Stronger team collaboration and shared product mindset.
    - 🗂️ Transparent backlog visible to the entire IT department.
    - 📄 Standardized user story and documentation process.
🛠 Tools & Technologies Used:
    🧠 ServiceNow – Platform for automation and workflow configuration.
    📋 Jira – For backlog tracking and sprint management.
    🎨 Figma – Used to design visuals and process diagrams.
    📊 Excel / Power BI – For dashboard and KPI tracking.
    📚 knowledge base/ Notion – For documentation and knowledge sharing.
